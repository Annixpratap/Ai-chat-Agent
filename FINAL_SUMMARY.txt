2. ✅ BACKEND API (TypeScript)
   Status: COMPLETE
   Endpoints:
   • POST /api/chat/message
     - Input: { message: string, conversationId?: string }
     - Output: { conversationId, userMessage, aiReply }
   • GET /api/chat/:conversationId
     - Output: { conversationId, messages[] }
   • GET /health (health check)
   
   Implementation:
   • 100% TypeScript with strict mode
   • Express.js server
   • CORS protected
   • Input validation at every layer
   • Error handling middleware

3. ✅ LLM INTEGRATION
   Status: COMPLETE
   Providers:
   • Anthropic Claude (claude-3-5-sonnet-20241022) - DEFAULT
   • OpenAI GPT-3.5-turbo
   • Configurable via LLM_PROVIDER env var
   
   Security:
   • API keys via environment variables only
   • No hardcoded secrets in repository
   • .env files in .gitignore
   
   Features:
   • System prompt with store knowledge
   • Full conversation history included for context
   • Error handling for API failures
   • Graceful fallback messages
   • Token limiting (max 500 tokens per response)
   • Context limiting (max 20 messages)

4. ✅ FAQ & DOMAIN KNOWLEDGE
   Status: COMPLETE
   Knowledge Base:
   • Shipping Policy (worldwide, standard, express, tracking)
   • Return/Refund Policy (30-day guarantee, processing)
   • Support Hours (Mon-Sat, email response time)
   • Product Info (earbuds, chargers, stands, protectors, USB hubs)
   • Support Contact (email address)
   
   Storage: Hardcoded in system prompt (MVP approach, can upgrade)

5. ✅ DATA PERSISTENCE
   Status: COMPLETE
   Database: SQLite (zero setup required)
   Schema:
   • conversations (id, createdAt, updatedAt)
   • messages (id, conversationId, sender, text, createdAt)
   • Indexed queries for performance
   • Foreign key constraints
   
   Session Recovery:
   • Conversation ID stored in browser localStorage
   • History fetched on page reload
   • Full conversation restored automatically

6. ✅ ROBUSTNESS & ERROR HANDLING
   Status: COMPLETE
   Input Validation:
   • Empty messages rejected (returns 400)
   • Long messages rejected (>2000 chars, returns 400)
   • Type checking everywhere
   
   Error Handling Layers:
   1. Route-level validation
   2. Service-level validation
   3. Database transaction safety
   4. LLM API error catching
   5. Network timeout handling (30s)
   6. Graceful fallback messages
   7. Error handler middleware
   
   Specific Cases Handled:
   • Empty message → "Message cannot be empty"
   • Message too long → "Message too long (max 2000)"
   • No API key → "Support system temporarily unavailable"
   • Rate limit → "Too many requests, please wait"
   • Timeout → "Request took too long"
   • Network error → "Network connection issue"
   • Invalid conversation ID → "Conversation not found"
   • Backend never crashes → All errors caught

================================================================================
                              QUICK START
================================================================================

1. Backend Setup:
   cd backend
   npm install
   cp .env.example .env
   # Edit .env: add ANTHROPIC_API_KEY or OPENAI_API_KEY
   npm run seed        # Initialize database
   npm run dev         # Start server (http://localhost:3000)

2. Frontend Setup (in new terminal):
   cd frontend
   npm install
   npm run dev         # Start dev server (http://localhost:5173)

3. Open Browser:
   http://localhost:5173
